# 设计规范：非标自动化选型计算工具

## 2026-07-09 设计接管说明

用户已明确不再自行设计 UI。本文档重新作为当前界面设计和实现的事实来源。后续若用户再提供设计稿，可作为新一轮迭代输入，但不阻塞当前开发。本节是最高优先级设计要求，若与后文历史页面清单冲突，以本节为准。

当前界面规则：

- 主界面只保留“选型计算”。
- 计算完成后在结果页直接导出报告。
- 删除用户可见的 PDF 覆盖矩阵、QA 覆盖检查、知识检索、内部参数库、独立报告导出入口。
- 厂家样本导入、字段映射、启停和删除只允许出现在计算结果页的“型号推荐”区域内，不做独立主页面或主导航入口。
- 删除界面中的 PDF 来源、页码、资料追溯等痕迹。
- 全部可见文案中文优先，不出现无必要英文。
- 桌面端采用三工作区：左侧计算对象列表、中间工况输入、右侧结果与风险。
- 设计目标是机械设计师日常高频使用的工程计算器，不做营销首页、装饰插画或大面积留白。

## 0. AI 使用说明

- 本文档定义产品的界面结构、视觉系统、组件、状态和交互规则。
- `Product-Spec.md` 是功能范围的事实来源；本文档不得新增 `Product-Spec.md` 没有定义的核心功能。
- 本产品是界面型 Windows 桌面离线应用；终端型与对话式 Agent 规格标记为 N/A。
- AI MUST 优先设计 P0 flows 和 P0 screens。
- AI MUST 覆盖默认、加载、空、错误、成功、禁用等关键状态。
- AI MUST 保持页面、组件、交互和视觉 tokens 的一致性。
- AI MUST 保证所有计算和报告导出界面都能展示公式、代入值、中间值、风险和用户确认动作，但不得展示 PDF 页码或内部资料来源。

## 1. 设计方向

### 1.0 产品形态

| 维度 | 方向 |
|---|---|
| 形态 | 界面型 Windows 桌面离线应用 |
| 本体 | 计算对象列表、工况表单、结果面板、风险面板、报告导出区 |
| 落地分轨 | 使用 §2 信息架构、§3 页面规格、§4 组件规格、§5 视觉系统、§8.3 响应式规则；§A 标 N/A |

### 1.1 设计目标

把“手算、查样本、凭经验判断”的碎片流程，收敛成一个高密度、可复算的工程选型计算器。用户打开应用后，应能直接选择计算对象，输入已知工况，查看过程分析和结论，并导出报告。

### 1.2 产品气质

| 维度 | 方向 |
|---|---|
| 语气 | 专业、直接、克制；少解释概念，多指出字段、风险和下一步 |
| 信息密度 | 紧凑；表单、表格、结果面板优先，不做营销式大标题和大留白 |
| 视觉风格 | 工程工具型 + 数据工作台；近似 CAD/ERP/技术后台的实用气质 |
| 情绪目标 | 用户感到可控、可信、能背锅；每个结论都有公式过程和风险说明 |

### 1.3 设计原则

- DP-001: 信息密度优先。常用操作和关键参数必须在首屏或固定面板可见，装饰元素让位给工况字段、计算步骤和风险。
- DP-002: 结果必须可复算。任何结论都要显示公式、代入值、中间值和用户输入快照，但界面不展示 PDF 页码或内部资料来源。
- DP-003: 安全系数必须显式确认。系统可以展示建议值，但输入或确认动作必须由用户完成，界面不得暗中套用默认安全系数。
- DP-004: 导出入口跟随计算结果。计算完成后在结果区直接导出 PDF 或 Excel，不做独立报告页。
- DP-005: 风险要比成功更显眼。安全系数不足、单位异常、速度/寿命/负载风险必须明确显示。
- DP-006: 结构稳定。左侧对象列表、中间工况输入、右侧结果风险区的空间分配保持一致，避免用户在不同模块里重新学习界面。

### 1.4 参考与反参考

**正向参考：**

| 参考产品 / 系统 | 具体借鉴属性 | 必须保留 |
|---|---|---|
| Microsoft Fluent 2 | 4px spacing ramp、清晰层级、适合 Windows 桌面生态的控件节奏 | Yes |
| IBM Carbon Data Table | 数据表的工具栏、搜索、筛选、排序、展开行和批量操作结构 | Yes |
| Grafana Dashboard | 状态面板、单位标注、说明信息、风险/异常定位方式 | Yes |
| GitHub Primer Forms / DataTable | 表单 label、验证反馈、表格可访问性和低认知负担 | Yes |

**反参考：**

- 营销型 SaaS 首页：大 hero、大插画、大留白会降低工程工具效率。

- 卡片堆卡片：嵌套卡片导致层级乱，工程数据难扫读。
- 纯 PDF 阅读器式界面：只搬运 PDF 文本不能完成选型计算，也不能满足 P0。

**外部参考确认：**

- Fluent 2 layout 强调间距、层级和 4px spacing ramp：https://fluent2.microsoft.design/layout
- Carbon Data Table 支持排序、展开、批量操作和表格工具栏：https://carbondesignsystem.com/components/data-table/usage/
- Grafana dashboard best practices 强调文档、单位和面板说明：https://grafana.com/docs/grafana/latest/visualizations/dashboards/build-dashboards/best-practices/

## 2. 信息架构

### 2.1 导航结构

采用固定三栏工作台：

| 区域 | 内容 | 规则 |
|---|---|---|
| 左侧工作区 | 计算对象列表和搜索 | 宽度约 270px，按对象名称、分类和描述搜索 |
| 中间工作区 | 工况参数、单位、安全系数确认和计算按钮 | 表单优先，字段成组排列，不展示来源页码 |
| 右侧工作区 | 计算结果、型号推荐、样本导入、样本库启停删除、报告导出 | 宽度约 360px；窄屏时结果区换到整行 |
| 顶部状态区 | 当前模块、数据库状态、加载/错误/完成状态 | 只显示业务状态，不放独立样本库、QA、知识库入口 |

### 2.2 页面清单

| 页面编号 | 页面名称 | 页面目的 | 关联流程 / 需求 | 优先级 |
|---|---|---|---|---|
| SCREEN-001 | 选型计算工作台 | 搜索计算对象、输入工况、安全系数确认、执行计算 | FLOW-001, REQ-002, REQ-003 | P0 |
| SCREEN-002 | 计算结果与过程分析 | 展示公式、代入、中间值、结论、风险、型号推荐和报告导出 | FLOW-001, FLOW-002, FLOW-003 | P0 |
| SCREEN-003 | 样本导入内联面板 | 在结果页内读取样本预览、编辑字段映射、确认入库 | FLOW-002, REQ-005 | P0 |
| SCREEN-004 | 样本库内联管理 | 在结果页内启用、停用、删除已导入样本库 | FLOW-002, REQ-005 | P0 |
| SCREEN-005 | 候选型号列表 | 在结果页内展示满足条件、不足原因并选择最终型号 | FLOW-002, REQ-006 | P0 |
| SCREEN-006 | 报告导出内联区 | 在结果页内导出 PDF 或 Excel 报告 | FLOW-003, REQ-008 | P0 |
| SCREEN-007 | 设置 | 数据库路径、备份、单位偏好、窗口布局 | 非功能需求 | P1 |
| SCREEN-012 | 设置 | 数据库路径、备份、单位偏好、窗口布局 | 非功能需求 | P1 |

### 2.3 页面关系

主路径一：`SCREEN-001` 选择对象并输入工况 -> `SCREEN-002` 查看过程和结论 -> `SCREEN-005` 匹配型号 -> `SCREEN-006` 导出报告。

主路径二：`SCREEN-002` 的型号推荐区 -> `SCREEN-003` 抽取与字段映射 -> `SCREEN-004` 管理样本库 -> `SCREEN-005` 使用推荐。

辅助路径：`SCREEN-007` 不阻塞 P0 使用；QA 和知识库仅作为开发内部能力，不进入用户主界面。

## 3. 页面规格

### SCREEN-001: 工作台首页 / PDF 覆盖矩阵

**页面目的：**  
让用户确认第一版覆盖根目录 PDF 的 23 个章节，并快速进入某一章节的计算或规则选型入口。

**主要操作：**  
搜索模块、打开章节、查看覆盖状态、进入最近使用模块。

**布局结构：**
1. 顶部区域：模块搜索、数据库状态、根目录 PDF 状态。
2. 主内容区：23 章覆盖矩阵表格，列为章节、实现形态、入口数量、来源页范围、状态、最近更新时间。
3. 右侧追溯区：当前选中章节的计算项、规则项、知识引用摘要。
4. 固定操作区：刷新覆盖矩阵、打开 QA 检查。

**内容层级：**

| 优先级 | 元素 | 说明 |
|---|---|---|
| 1 | 23 章覆盖状态 | 防止范围缩水，是第一版成败的主控面板 |
| 2 | 模块搜索与最近使用 | 日常高频入口 |
| 3 | PDF 来源与解析状态 | 说明根目录 PDF 是否已解析 |

**使用的组件：**

| 组件编号 | 组件名称 | 用途 |
|---|---|---|
| CMP-001 | AppShell | 全局工作台框架 |
| CMP-002 | ModuleCoverageMatrix | 23 章覆盖矩阵 |
| CMP-008 | SourcePageLink | 来源页码入口 |
| CMP-007 | RiskBadge | 覆盖缺口、异常状态提示 |

**必需状态：**
- 默认：显示 23 章列表，未选中章节时右侧显示最近使用摘要。
- 加载：读取覆盖矩阵时表格行使用骨架状态，顶部显示轻量进度。
- 空状态：首次启动未解析 PDF 时，显示“导入根目录 PDF”入口。
- 错误：PDF 不存在或解析失败时显示文件路径和重试操作。
- 成功：章节状态显示已实现/待实现/部分实现，入口可点击。
- 禁用：数据库不可写时禁用修改类操作，只允许查看。

**响应式表现：**
- 桌面端：三栏布局。
- 平板端：右侧追溯区折叠为可固定抽屉。
- 移动端：左侧导航变为抽屉，覆盖矩阵横向滚动；移动端不是主目标，只保证可查阅。

**可访问性说明：**
- 覆盖状态不能只用颜色区分，必须配文字。
- 表格列头支持键盘排序。
- 搜索框有明确 label。

### SCREEN-002: 选型/计算向导

**页面目的：**  
让用户按模块输入工况参数、安全系数和单位，完成计算或规则选型。

**主要操作：**  
填写字段、选择单位、确认安全系数、执行计算。

**布局结构：**
1. 顶部区域：模块标题、章节来源、计算/规则类型、案例保存状态。
2. 主内容区：分组表单，左侧为工况输入，右侧为默认参数选择和单位设置。
3. 右侧追溯区：字段说明、PDF 来源页码、建议值来源、安全系数提示。
4. 固定操作区：计算、重置、保存草稿。

**内容层级：**

| 优先级 | 元素 | 说明 |
|---|---|---|
| 1 | 必填工况字段 | 负载、速度、行程、效率、安全系数等 |
| 2 | 单位与默认参数来源 | 防止单位错和默认值误用 |
| 3 | 字段说明和来源页码 | 帮助新人成本低地理解输入 |

**使用的组件：**

| 组件编号 | 组件名称 | 用途 |
|---|---|---|
| CMP-003 | CalculationForm | 动态渲染模块字段 |
| CMP-004 | UnitInput | 数值 + 单位组合输入 |
| CMP-005 | SafetyFactorInput | 安全系数输入与确认 |
| CMP-006 | TracePanel | 字段说明和来源 |
| CMP-008 | SourcePageLink | PDF 来源 |

**必需状态：**
- 默认：表单为空，必填字段清晰标识。
- 加载：执行计算时按钮显示进行中，表单不整体跳动。
- 空状态：未选择模块时显示模块选择入口。
- 错误：定位到具体字段，说明非法值和单位问题。
- 成功：进入结果状态或打开结果面板。
- 禁用：安全系数未确认时禁用计算按钮，并在字段旁说明原因。

**响应式表现：**
- 桌面端：表单和右侧追溯区并排。
- 平板端：追溯区在右侧抽屉。
- 移动端：单列表单，字段说明折叠在每组字段下。

**可访问性说明：**
- 每个输入必须有 label、单位、错误信息关联。
- 数值输入支持键盘操作。
- 公式和单位说明可复制。

### SCREEN-003: 计算结果与过程分析

**页面目的：**  
展示可复算的计算过程、结论、风险提示，并作为保存案例、匹配型号、导出报告的入口。

**主要操作：**  
查看过程、保存案例、匹配型号、导出报告。

**布局结构：**
1. 顶部区域：结果状态、模块、运行时间、公式版本。
2. 主内容区：结果摘要、关键中间值、公式步骤、代入过程。
3. 右侧追溯区：风险列表、来源页码、默认参数来源、推荐入口。
4. 固定操作区：保存案例、复制案例、匹配型号、导出。

**内容层级：**

| 优先级 | 元素 | 说明 |
|---|---|---|
| 1 | 结论与风险 | 用户先判断能不能用 |
| 2 | 公式步骤与中间值 | 用户判断为什么这样算 |
| 3 | 来源和版本 | 用户追溯 PDF、参数库、样本库 |

**使用的组件：**

| 组件编号 | 组件名称 | 用途 |
|---|---|---|
| CMP-006 | TracePanel | 公式、代入、来源、风险 |
| CMP-007 | RiskBadge | 风险状态 |
| CMP-008 | SourcePageLink | PDF 来源 |
| CMP-013 | ReportExportDialog | 导出入口 |

**必需状态：**
- 默认：显示最近一次运行结果。
- 加载：计算中展示局部进度，不清空用户输入。
- 空状态：没有运行结果时引导返回计算表单。
- 错误：公式执行失败时显示失败步骤和输入快照。
- 成功：展示可保存、可导出的完整结果。
- 禁用：无结果时禁用匹配型号和导出。

**响应式表现：**
- 桌面端：过程分析主区 + 右侧风险/来源区。
- 平板端：风险/来源区可固定或折叠。
- 移动端：结果摘要、风险、过程分段纵向排列。

**可访问性说明：**
- 风险信息用图标、文字和色彩三重表达。
- 公式块使用等宽字体，支持复制。
- 导出、保存、删除等操作有焦点态和确认态。

### SCREEN-004: 案例库

**页面目的：**  
让用户复用历史选型计算，减少重复输入和手算。

**主要操作：**  
搜索案例、打开案例、复制为新案例、删除案例。

**布局结构：**
1. 顶部区域：搜索、模块筛选、时间筛选。
2. 主内容区：案例表格。
3. 右侧详情区：选中案例的输入快照、结果摘要、风险、推荐型号。
4. 固定操作区：打开、复制、删除。

**必需状态：**
- 默认：显示最近案例。
- 加载：读取数据库时表格骨架。
- 空状态：无案例时显示“新建计算”入口。
- 错误：数据库读取失败，显示备份路径和重试。
- 成功：打开或复制案例后进入计算向导。
- 禁用：数据库只读时禁用保存和删除。

### SCREEN-005: 厂家样本库

**页面目的：**  
管理厂家 PDF/Excel/CSV 样本库版本和启用状态。

**主要操作：**  
导入样本、启用/禁用、查看导入记录、删除样本库。

**布局结构：**
1. 顶部区域：导入按钮、样本类型筛选、启用状态筛选。
2. 主内容区：样本库列表。
3. 右侧详情区：文件来源、导入时间、字段数量、失败记录。
4. 固定操作区：重新导入、禁用、删除。

**必需状态：**
- 默认：显示样本库列表。
- 加载：读取样本库。
- 空状态：提示导入第一个厂家 PDF 样本。
- 错误：样本库损坏或字段缺失。
- 成功：样本库可参与型号推荐。
- 禁用：禁用样本库时明确标记不参与推荐。

### SCREEN-006: 样本导入向导

**页面目的：**  
把 PDF/Excel/CSV 样本从文件变成经过确认的结构化型号参数。

**主要操作：**  
选择文件、预览抽取、映射字段、确认导入。

**布局结构：**
1. 顶部区域：四步进度：选择文件 -> 抽取预览 -> 字段映射 -> 确认导入。
2. 主内容区：抽取表格或字段映射表。
3. 右侧追溯区：PDF 页码、置信度、单位识别、错误行。
4. 固定操作区：上一步、下一步、确认导入、取消。

**必需状态：**
- 默认：等待选择文件。
- 加载：解析文件，显示当前文件名和进度。
- 空状态：无可抽取文本时提示 OCR 或手动录入。
- 错误：格式错误、缺必填字段、单位无法识别。
- 成功：显示导入成功数量、失败数量和失败原因。
- 禁用：字段映射未确认时禁用导入。

### SCREEN-007: 型号推荐结果

**页面目的：**  
显示候选型号、匹配依据和未匹配原因，允许用户手动选择最终型号。

**主要操作：**  
筛选候选、查看匹配条件、查看失败原因、选择最终型号。

**布局结构：**
1. 顶部区域：需求参数摘要、样本库范围、筛选器。
2. 主内容区：候选型号表格。
3. 右侧详情区：选中型号参数、满足条件、失败条件、来源页码。
4. 固定操作区：确认型号、返回结果、导出报告。

**必需状态：**

## 6. 交互与反馈规则

### 6.1 全局交互规则

| 模式 | 规则 |
|---|---|
| 主操作 | 固定在页面右上或底部操作区；每页最多一个主按钮 |
| 次要操作 | 使用普通按钮或图标按钮，靠近相关内容 |
| 危险操作 | 删除案例、删除样本库、删除参数必须二次确认 |
| 保存行为 | 案例、参数、样本导入均为显式保存；历史计算快照不可被参数变更自动改写 |
| 导航 | 离开未保存表单前提示；返回时保留当前输入 |
| 弹窗 / 抽屉 | 短确认用弹窗；长表单、来源追溯、失败详情用右侧抽屉 |
| 表格 | 支持排序、筛选、搜索；大表格避免分页跳转优先虚拟滚动 |
| 安全系数 | 未输入或未确认时禁止计算，错误定位到字段 |
| PDF 导入 | 未完成字段映射和确认时禁止写入样本库 |

### 6.2 反馈规则

| 场景 | UI 反馈 | 文案规则 |
|---|---|---|
| 成功 | 轻量 toast + 状态文字；不使用大面积绿色 | 说明完成对象，例如“案例已保存” |
| 错误 | 字段内联错误 + 页面级错误摘要 | 指出字段、原因、处理方式 |
| 警告 | 黄色风险条或 RiskBadge | 说明风险，不替用户做工程判断 |
| 空状态 | 图标 + 原因 + 下一步按钮 | 不写鼓励话，直接给入口 |
| 加载 | 局部骨架、进度条或按钮 loading | 显示正在处理的文件/模块 |

### 6.3 动效规则

- 动效只用于状态切换、抽屉展开、toast 出入、表格行展开。
- 动效时长 120-180ms，缓动使用 ease-out。
- 禁止装饰性循环动画。
- 尊重 reduced motion 设置；关闭后只保留必要状态变化。

## 7. UX 文案与内容规则

### 7.1 文案语气

清晰、直接、工程化。界面不扮演老师，不说“恭喜”“太棒了”。错误和风险要具体，成功提示要短。

### 7.2 按钮文案规则

- 主按钮用动词 + 对象：`计算`、`保存案例`、`匹配型号`、`确认导入`、`导出 PDF`、`导出 Excel`。
- 次按钮用明确动作：`复制案例`、`重置`、`返回上一步`、`打开来源`。
- 危险按钮必须带对象：`删除案例`、`删除样本库`、`删除参数`。
- 图标按钮必须有 tooltip。

### 7.3 表单文案规则

- label 使用工程术语，必要时在 helper text 解释。
- placeholder 只给输入格式，例如 `例如 12.5`，不写长句说明。
- 单位写在输入框右侧或下拉中，不写进 label。
- 错误文案格式：`字段 + 问题 + 修正动作`，例如“安全系数未确认，请输入或确认本次计算使用值”。

### 7.4 空状态文案规则

- 空状态先说原因，再给下一步。
- 例：`还没有样本库。导入厂家 PDF 后才能进行型号推荐。`
- 不使用鼓励式、营销式或泛泛解释。

## 8. 可访问性与响应式要求

### 8.1 可访问性目标

基础目标按 WCAG 2.2 AA 的核心要求执行：键盘可用、焦点可见、文本对比度合格、表单错误可感知、状态不只依赖颜色。

### 8.2 必需可访问性规则

- [ ] 所有可交互元素可通过键盘访问。
- [ ] 焦点状态清晰可见。
- [ ] 表单字段有明确 label。
- [ ] 错误信息与对应字段关联。
- [ ] 不只依赖颜色表达状态。
- [ ] 正文和背景有足够对比度。
- [ ] 图标按钮有可读标签或 tooltip。
- [ ] 页面标题和区块标题结构清晰。
- [ ] 表格列头可被辅助技术识别。
- [ ] 公式、步骤、来源页码可复制。

### 8.3 响应式规则

| 断点 / 设备 | 规则 |
|---|---|
| 移动端 <= 768px | 单列布局；左侧导航变抽屉；右侧追溯区变底部抽屉；表格允许横向滚动；不作为主要生产形态 |
| 平板端 769-1199px | 左侧导航可折叠；右侧追溯区默认折叠；主工作区优先显示表单和表格 |
| 桌面端 >= 1200px | 默认三栏工作台；推荐目标宽度 1366-1920px；右侧追溯区固定 360px |
| 宽屏 >= 1600px | 主工作区允许双列字段分组；不放大字号，只增加可见列和面板宽度 |

## 9. 实现说明

- 技术栈按 `DEV-PLAN.md`：Tauri + React + Vite + TypeScript + Rust + SQLite。
- 前端样式优先使用 CSS variables + 轻量组件封装；如引入 Tailwind，只用于 token 映射，不让工具类替代组件语义。
- 图标使用 lucide-react。
- P0 默认浅色主题；暗色模式不进入 P0，除非后续需求明确要求。
- 不需要品牌资产；第一版使用文字产品名和简洁应用图标即可。
- 禁止营销首页，启动后直接进入工作台首页 / PDF 覆盖矩阵。
- 禁止把 PDF 正文当作主要 UI；PDF 内容必须转成计算向导、规则向导或知识引用。
- 数据密集页面优先使用表格、分组表单、侧边追溯面板，不使用大号装饰卡片。
- 所有固定格式 UI，如工具栏、表格行、图标按钮、状态徽标、单位输入，必须定义稳定尺寸。
