# UI 设计接管说明

用户已确认不再自行设计 UI。当前第一版界面由开发方直接负责设计和实现。

## 当前执行方向

- 主界面只保留“选型计算”。
- 采用左侧计算对象列表、中间工况输入、右侧结果与风险的工程工作台布局。
- 计算完成后在结果页直接导出 PDF 或 Excel。
- 不再展示 PDF 覆盖矩阵、QA 覆盖检查、知识检索、内部参数库、独立报告导出页。
- 不在界面展示 PDF 页码、内部资料名或来源痕迹。
- 风格保持中文、紧凑、克制、工程工具型。

---

以下内容仅保留为后续如果用户重新提供设计稿时的交接参考，不作为当前阻塞条件。

## 1. 可以用什么工具

- Figma
- MasterGo
- 墨刀
- Axure
- PPT
- 图片标注
- PDF 标注
- 手绘草图拍照

优先级：可编辑设计稿 > 带尺寸标注的图片 > 普通截图 > 手绘草图。

## 2. 至少要给哪些页面

第一版只需要这些界面：

1. 选型计算首页
2. 计算对象列表和搜索
3. 参数输入表单
4. 计算结果与过程分析
5. 结果页内导出报告区域
6. 空状态、错误状态、计算中状态

不需要再设计 PDF 覆盖矩阵、QA 覆盖检查、知识检索、内部参数库、独立报告导出页。

## 3. 每个页面最好标清楚什么

- 页面宽度：例如 1366、1440、1920。
- 左右边距和主要区域宽度。
- 字体大小和字重。
- 颜色：主色、背景色、边框色、错误/警告/成功色。
- 按钮样式：主按钮、普通按钮、禁用按钮。
- 输入框样式：默认、聚焦、错误、禁用。
- 表格或列表样式。
- 结果区公式、结论、风险提示的显示顺序。
- 导出报告区域的位置和样式。

如果不会标尺寸，直接写“照这个截图比例做”也可以，但实现时我会按桌面端 1366px 到 1920px 做自适应。

## 4. 设计稿里必须出现真实业务内容

别放 lorem ipsum，也别只画空壳。至少放这些示例内容：

- 选型对象：伺服电机、步进电机、气缸、真空吸附、同步带、滚珠丝杠、直线导轨。
- 输入字段：负载质量、行程、速度、加速时间、摩擦系数、效率、安全系数。
- 计算结果：需求转矩、需求转速、推力、吸附力、寿命、风险提示。
- 导出格式：PDF、Excel。

界面设计要为这些真实文字留够空间，不能只适配短标题。

## 5. 交付给我的方式

任选一种：

- 发 Figma / MasterGo / 墨刀链接。
- 把设计稿导出成 PNG 或 PDF 放到项目根目录。
- 用 PPT 画完后放到项目根目录。
- 手绘后拍照，图片放到项目根目录。

文件名建议：

- `UI-选型计算-首页.png`
- `UI-参数输入.png`
- `UI-计算结果.png`
- `UI-导出报告.png`

## 6. 我拿到设计稿后会做什么

1. 对照页面清单检查是否缺关键状态。
2. 把设计稿拆成组件：导航、模块列表、参数表单、结果卡片、导出区域、风险提示。
3. 先实现一个核心页面闭环。
4. 跑桌面端截图检查文字是否溢出、按钮是否挤压、结果是否可读。
5. 再替换其他页面的临时样式。
