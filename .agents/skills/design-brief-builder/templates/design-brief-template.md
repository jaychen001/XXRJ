## 0. AI 使用说明

- 本文档定义产品的界面结构、视觉系统、组件、状态和交互规则。

## 写作要点

1. **设计方向**：设计目标一句；产品气质 4 维（语气 / 密度 / 视觉风格 / 情绪）；设计原则可操作；参考产品拆 load-bearing 属性（「像 Stripe」是症状不是规格），反参考当边界 fence。
2. **信息架构**：导航结构 + 页面清单（每页关联 FLOW / REQ、标 P0 / P1）+ 页面关系。
3. **页面规格**：每个有设计决策空间的页面写布局 / 内容层级 / 用的组件 / 六态 / 响应式；纯后端页跳过。
4. **组件规格**：核心组件的变体 + 七态 + 内容规则 + 可访问性；组件编号 CMP 供页面引用。
5. **视觉系统**：色彩 / 字体 / 间距 / 形状用 token 表；具体值定不下来标 [待补充] 交设计工具，但方向要明确。
6. **交互与反馈**：全局交互规则 + 五场景反馈 + 动效克制还是明显。
7. **UX 文案**：语气 + 按钮 / 表单 / 空态文案规则。
8. **可访问性与响应式**：a11y 逐条勾、对比度和动效偏好表态、响应式断点逐档。
9. **实现说明**：技术栈偏好、暗色、品牌资产、必用 / 避免的设计模式。
10. **假设与待确认**：定不下来的写假设或待确认，不凭空补。
11. **跨文档一致**：SCREEN / CMP 编号呼应 Product Spec 的 FLOW / REQ，两边 P0 一致。
12. **形态分轨**：先认形态填 §1.0。界面型走 §2 §3 §5 §8.3；终端型和对话式 Agent 走 §A，§5 改 A.5 终端口径，无关段标 N/A。Agent 形态的授权与透明度 A.4 必填，别留空。
