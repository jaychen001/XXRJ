# 产品需求规范：非标自动化选型计算工具

## 0. AI 使用说明

- 本文档是产品功能、范围、行为和验收标准的事实来源。
- AI MUST 优先实现 P0。
- AI MUST NOT 实现“不在本版本范围”中明确排除的内容。
- AI MUST 根据“验收标准”判断功能是否完成。
- 如果信息不明确，AI MUST 使用“假设”中的假设；如果仍无法判断，应记录到“待确认问题”，而不是自行扩展需求。

### 0.1 2026-07-08 用户测试修正

本节是当前最高优先级需求，若与后文旧表述冲突，以本节为准。旧版本里“PDF 覆盖矩阵、QA 覆盖检查、知识检索、内部参数库、来源页码追溯”等内容属于开发期资料或历史实现，不再作为用户可见的一版产品功能。

- P0 只保留两个用户主入口：`选型计算`、以及计算完成后的`导出报告`。
- 报告导出必须放在计算结果页内，不做独立“报告导出”主页面。
- 用户界面必须中文优先，删除无必要英文、Phase、QA、PDF、知识库、覆盖矩阵、内部参数库等工程开发用语。
- 选型结果只展示公式、代入过程、中间值、结论、风险提示和可导出报告，不展示 PDF 页码、原始资料名或“根据某 PDF”的痕迹。
- 第一版仍要覆盖既定选型目录中的产品类型，但公式来源要以公开工程公式、厂家技术手册、行业通用计算方法和内部经验复判为依据，不把根目录 PDF 暴露为产品来源。
- 参数表单必须按设计师实际已知工况组织，而不是机械照搬资料目录。每个模块要先定义“用户通常已知什么”，再决定输入字段。
- 安全系数由用户手动输入并确认；系统只做风险提示，不替用户背工程判断。
- “内部参数库”页面删除。默认参数可以作为代码内置建议或模块默认值存在，但不在主界面做独立维护入口。
- 厂家样本库和型号推荐能力可以作为后续增强保留在底层，但第一版主界面不暴露独立厂家库入口；型号推荐只在计算结果上下文中出现。
- UI 由开发方直接负责设计和实现，不再等待用户提供设计稿。界面方向采用中文优先、工程工具型、高信息密度、计算结果可快速扫读的桌面工作台。

### 0.2 2026-07-09 公式输入重构状态

已开始把高频模块从“资料字段”改为“设计师实际已知工况字段”。本轮已落地：

- 伺服/步进：补充外部阻力、垂直负载系数，扭矩计算覆盖摩擦、加速、重力和外部阻力。
- 滚珠丝杠：补充外部轴向力、垂直负载系数、支撑跨距、底径、支撑方式系数、额定动载荷和目标行走寿命，输出临界转速和寿命风险。
- 气缸：补充外部阻力、垂直负载系数和有效面积系数，缸径反算覆盖推程/回程面积差。
- 手指气缸 / 滑台气缸 / 旋转气缸：补充候选规格参数，分别输出夹持力与手指力矩、推力与偏载/动能、旋转扭矩与动能余量。
- 真空吸附：补充姿态修正系数，吸附力计算覆盖搬运姿态和有效吸附率。
- 直线导轨：补充目标行走寿命，输出额定寿命估算和寿命风险。
- 机器人、拖链、传感器、材料、机加工、热处理与表面处理、常用五金件：规则型入口补齐工况问题、推荐结论、判断依据和风险提示，结果页不展示 PDF 来源。

### 0.3 2026-07-09 UI 设计接管

用户已确认不再自行设计 UI。第一版界面由开发方按以下标准直接落地：

- 主界面仍只保留“选型计算”，不恢复任何旧工具页入口。
- 布局采用左侧计算对象列表、中间工况输入、右侧结果与风险的工程工作台结构。
- 首屏要能完成“选对象 -> 填参数 -> 确认安全系数 -> 计算 -> 导出报告”的闭环。
- 视觉风格保持克制、紧凑、中文、面向机械设计师日常使用；不做营销页、大装饰图、大留白。
- 所有按钮、输入框、结果、风险、空状态和错误状态必须有完整中文文案。

## 1. 产品上下文

### 1.1 产品摘要

非标自动化选型计算工具是一款 Windows 桌面离线应用，用于把根目录 PDF 中所有非标机械选型内容从“翻 PDF、查样本、手算、靠经验判断”收敛成结构化选型/计算向导。用户输入工况参数后，系统输出过程分析、计算步骤、风险提示和选型结论，并可结合导入的厂家 PDF/Excel/CSV 样本库给出候选型号推荐。

第一版服务内部使用，不做公开商业 SaaS。

### 1.2 用户问题

内部机械设计人员在日常非标自动化设计中，需要频繁使用根目录 PDF 中的电机、丝杆、同步带、V 带、齿轮、链条、减速机、直线模组、凸轮分割器、制动器/离合器、机器人、气动执行元件、气动控制、直线导轨、直线轴承、滚动轴承、联轴器、拖链、传感器、材料、机加工、热处理与表面处理、常用五金件等选型内容。现状依赖 PDF 笔记、厂家样本、Excel、手算和个人经验，容易出现以下问题：

- 同一类工况重复计算，过程无法复用。
- 计算公式、单位换算、安全系数和效率项容易漏。
- 新人难以判断“为什么这样选”，只看到结论不敢背锅。
- 厂家样本参数与内部经验参数分散，型号推荐缺少统一入口。

### 1.3 目标用户

| 用户类型 | 描述 | 核心需求 |
|---|---|---|
| 主用户 | 用户本人和小组内非标机械设计成员 | 快速完成常用机构选型计算，并能看到过程分析和结论 |
| 次级用户 | 小组内经验较少的机械设计人员 | 按向导输入工况，理解计算依据，减少漏项 |

### 1.4 核心价值

把高频选型计算从“散落资料 + 个人经验”变成“可复算、可追溯、可复用、可导出”的本地工作流。

### 1.5 成功标准

| 判断标准 | 目标 / 信号 |
|---|---|
| PDF 选型覆盖 | P0 覆盖根目录 PDF 中全部选型内容；有公式的做计算向导，只有经验判断的做选型向导 |
| 结果可信 | 每次计算都输出公式、代入值、单位、关键中间值、结论和风险提示 |
| 离线可用 | 不联网时可创建案例、运行计算、查看历史、导入本地 PDF/Excel/CSV 样本库、导出报告 |
| 可复用 | 已保存案例可复制为新案例，修改参数后重新计算 |
| 型号推荐 | 导入厂家样本库后，系统可按计算结果和内部参数过滤候选型号 |

## 4. 用户流程

### FLOW-001: 完成一次选型计算

**关联任务：** TASK-001, TASK-002, TASK-003  
**优先级：** P0  
**目标：** 让用户从选择计算类型到拿到过程分析和结论。

**入口：**  
用户打开应用首页，点击某个计算模块，例如伺服电机、步进电机、气缸、真空、同步带、滚珠丝杠。

**主路径：**
1. 用户选择选型对象。
2. 系统展示该对象的工况输入表单和必要参数说明。
3. 用户输入负载、速度、行程、加速时间、摩擦系数、效率、安全系数等参数。
4. 系统进行单位校验和字段校验。
5. 用户点击“计算”。
6. 系统输出计算过程、关键中间值、结论和风险提示。
7. 用户保存案例或复制为新案例。

**分支路径：**
- 用户缺少参数时，可选择默认内部参数；系统必须标记该值来自默认值。
- 用户修改任意输入后，系统允许重新计算并保留计算版本。
- 用户不保存时，系统允许作为临时计算查看结果。

**边界情况：**
- 必填字段为空时，禁止计算并定位到字段。
- 单位不一致时，系统必须换算或提示用户选择单位。
- 计算结果超出常用范围时，系统必须给出风险提示，而不是只给结论。

**完成状态：**
用户看到一份可复算的计算结果：输入参数、默认参数来源、公式、代入过程、中间值、结论、风险、推荐候选型号。

### FLOW-002: 导入厂家样本库并推荐型号

**关联任务：** TASK-004, TASK-005  
**优先级：** P0  
**目标：** 让用户把本地厂家 PDF/Excel/CSV 型号参数导入后，用计算结果筛选候选型号。

**入口：**  
第一版主界面不暴露独立样本库入口；样本导入能力保留为后台/增强能力，型号推荐入口只在计算结果页出现。

**主路径：**
1. 用户选择样本类型，例如伺服电机、气缸、导轨。
2. 用户选择 PDF、Excel 或 CSV 文件，其中 PDF 是第一优先导入格式。
3. 如果是 PDF，系统抽取文本、表格、样本位置和疑似型号参数，并展示抽取置信度。
4. 如果是 Excel/CSV，系统读取表头和行数据。
5. 系统展示字段映射界面。
6. 用户把抽取列或文本字段映射到系统字段，例如型号、额定转矩、最高转速、缸径、行程、负载能力。
7. 系统校验字段类型和单位。
8. 用户确认导入。
9. 系统保存样本库记录。
10. 用户完成计算后，在结果页点击“匹配型号”。
11. 系统按计算结论、内部规则和样本参数过滤候选型号，展示满足条件、不足原因，并允许选择最终型号。

**分支路径：**
- 字段缺失时，系统提示缺少哪些字段，并允许取消导入。
- 单位无法识别时，系统要求用户手动选择单位。
- PDF 抽取置信度低或页面为扫描图时，系统提示需要 OCR 或手动录入，不得静默导入。
- 用户可禁用或删除某个样本库版本。

**边界情况：**
- 同名型号重复导入时，系统提示覆盖、跳过或作为新版本导入。
- 文件格式错误时，系统不写入数据。
- PDF 字段映射未经用户确认时，系统不写入型号库。

**完成状态：**
样本库被保存，用户可在计算结果中看到候选型号、匹配依据和未匹配原因。

### FLOW-003: 导出计算报告

**关联任务：** TASK-007  
**优先级：** P0  
**目标：** 让用户把计算依据和结论交给评审、采购或电气沟通。

**入口：**  
用户在计算结果页点击导出。

**主路径：**
1. 用户选择导出格式：PDF 或 Excel。
2. 系统生成报告预览。
3. 用户确认导出路径。
4. 系统写出文件。

**分支路径：**
- 如果报告缺少型号推荐，仍允许导出计算报告。
- 如果案例未保存，导出前提示是否保存案例。

**边界情况：**
- 导出路径无权限时，提示用户更换路径。
- 文件已存在时，提示覆盖或另存。

**完成状态：**
用户获得一份包含输入、公式、过程、结论、风险和候选型号的本地报告文件。

## 5. 功能需求

### REQ-001: 计算模块导航

**优先级：** P0  
**关联任务：** TASK-001  
**关联流程：** FLOW-001  

**用途：**  
让用户快速进入某一类选型计算。

**行为：**  
系统按根目录 PDF 目录和选型类别展示模块，覆盖电机、丝杆、同步带、V 带、齿轮、链条、减速机、直线模组、凸轮分割器、制动器/离合器、机器人、气动执行元件、气动控制、直线导轨、直线轴承、滚动轴承、联轴器、拖链、传感器、材料、机加工、热处理与表面处理、常用五金件。

**规则：**
- MUST 支持按模块名称搜索。
- MUST 展示每个模块的适用场景摘要。
- SHOULD 支持最近使用模块置顶。

**输入：**

| 字段 | 类型 | 必填 | 校验规则 |
|---|---|---:|---|
| 模块搜索词 | 文本 | No | 长度不超过 50 |

**输出 / 结果：**
- 模块列表。
- 进入模块后的工况输入表单。

**状态：**
- 默认：展示全部模块。
- 加载：本地模块元数据读取中显示轻量加载状态。
- 空状态：搜索无结果时提示清空搜索。
- 错误：模块配置损坏时提示恢复默认配置。
- 成功：用户进入对应计算模块。

**验收标准：**
- [ ] AC-001: Given 应用已打开, when 用户点击“伺服电机”, then 系统进入伺服电机计算表单。
- [ ] AC-002: Given 用户输入“气”, when 搜索执行, then 系统显示气缸、真空、电磁阀等相关模块。

### REQ-002: 选型计算引擎

**优先级：** P0  
**关联任务：** TASK-001, TASK-002  
**关联流程：** FLOW-001  

**用途：**  
以确定性公式和规则完成选型计算。

**行为：**  
用户输入工况参数后，系统根据模块公式计算中间值和结论。计算必须保留完整步骤，不能只返回最终数值。

**规则：**
- MUST 所有计算公式代码化，不能依赖 AI 或自由文本推理。
- MUST 输出公式、单位、代入值、中间值、最终结论。
- MUST 标记每个默认值来源：系统默认、内部参数库、样本库、用户输入。
- MUST 将安全系数作为用户手动输入字段；系统可以显示来自根目录 PDF 或内部参数库的建议值，但不得自动替用户套用。
- MUST 支持单位换算，至少覆盖 mm/m、kg/N、s/min、rpm/rps、W/kW。
- MUST 对风险项输出提示，例如安全系数不足、速度超常用范围、惯量比过大、效率缺失。
- SHOULD 支持同一计算模块的低档/中档/高档计算模式，但 P0 可先实现常用模式。

**输入：**

| 字段 | 类型 | 必填 | 校验规则 |
|---|---|---:|---|
| 选型模块 | 枚举 | Yes | 必须来自系统模块 |
| 工况参数 | 数值对象 | Yes | 按模块字段校验 |
| 单位 | 枚举 | Yes | 必须来自系统单位表 |
| 安全系数 | 数值 | Yes | 大于 0；默认不自动计算，用户必须确认或输入 |
| 默认参数选择 | 枚举 | No | 系统默认 / 内部参数库 / 手动输入 |

**输出 / 结果：**
- 过程分析。
- 计算结论。
- 风险提示。

系统将 PDF 笔记抽取为本地文本索引，用户可搜索关键词并打开相关页码或条目。

**规则：**
- MUST 不把知识库检索结果当作计算结果。
- MUST 计算公式以代码实现为准。
- SHOULD 在计算步骤旁显示相关知识条目引用。

**输入：**

| 字段 | 类型 | 必填 | 校验规则 |
|---|---|---:|---|
| 搜索词 | 文本 | Yes | 1-100 字符 |

**输出 / 结果：**
- 知识条目列表。
- 来源页码或来源文件。

**状态：**
- 默认：显示搜索框。
- 加载：检索中。
- 空状态：无结果。
- 错误：索引损坏。
- 成功：显示结果。

**验收标准：**
- [ ] AC-027: Given 用户搜索“惯量比”, when 知识库索引存在相关内容, then 系统显示相关条目和来源页码。

### AI 能力规格

| AI 功能 | 能力类型 | 质量条 | 触发方式 | 不确定时 | 服务降级 |
|---|---|---|---|---|---|
| 暂不作为 P0 | 无 | 核心计算不使用 AI | 无 | 无 | 无 |

**AI 护栏（绝不能做）：**
- AI MUST NOT 直接给出最终选型计算结论。
- AI MUST NOT 替代确定性公式引擎。
- AI MUST NOT 自动修改内部参数库或样本库。
- AI MAY 在后续版本用于解释公式、辅助检索知识库、提示缺失参数，但输出必须可追溯到公式和数据源。

## 6. 数据模型

### 6.1 核心实体

| 实体 | 描述 | 关键字段 |
|---|---|---|
| CalculationModule | 计算模块 | id, name, category, description, version, fields |
| CalculationCase | 计算案例 | id, name, module_id, inputs, defaults_snapshot, result, risk_flags, selected_model, created_at, updated_at |
| CalculationRun | 单次计算记录 | id, case_id, module_id, input_snapshot, formula_version, result_snapshot, created_at |
| InternalParameter | 内部参数 | id, name, category, value, unit, scenario, source, enabled |
| VendorLibrary | 厂家样本库 | id, name, component_type, source_file, source_format, version_name, imported_at, enabled |
| VendorImportJob | 样本导入任务 | id, library_id, source_file, source_format, status, extracted_preview, confidence, created_at |
| VendorModel | 厂家型号 | id, library_id, model_name, brand, series, parameters, normalized_parameters, source_page |
| RecommendationResult | 型号推荐结果 | id, run_id, model_id, matched_rules, failed_rules, score |
| ReportExport | 报告导出记录 | id, case_id, run_id, format, path, exported_at |
| KnowledgeSource | 知识来源 | id, source_type, file_path, title, imported_at |
| KnowledgeEntry | 知识条目 | id, source_id, page, title, content, tags |

### 6.2 实体关系

| 关系 | 描述 |
|---|---|
| CalculationCase belongs to CalculationModule | 一个案例对应一个计算模块 |
| CalculationCase has many CalculationRun | 一个案例可多次修改参数并重新计算 |
| VendorLibrary has many VendorModel | 一个样本库包含多个型号 |
| VendorLibrary has many VendorImportJob | 一个样本库可对应多次导入任务 |
| CalculationRun has many RecommendationResult | 一次计算可生成多个候选型号 |
| KnowledgeSource has many KnowledgeEntry | 一个 PDF 或文档来源包含多个知识条目 |

### 6.3 数据规则

- 用户创建的案例、参数、样本库必须可删除。
- 删除案例不删除全局内部参数和厂家样本库。
- 删除样本库必须删除该库下的 VendorModel，并影响后续推荐，不应改写历史 CalculationRun 的结果快照。
- PDF 导入结果必须保存来源页码或来源区域，便于用户追溯。
- 修改内部参数不应自动改写历史案例和历史计算记录。
- 所有计算运行必须保存输入快照、默认参数快照和公式版本。
- 所有本地数据默认存储在本机，不依赖云端。
- 第一版默认无账号权限系统，所有本机使用者共享同一数据库；是否要加本地密码待确认。

## 10. 假设与待确认问题

### 10.1 假设

| 编号 | 假设 | 假设依据 | 错误风险 |
|---|---|---|---|
| ASM-001 | 第一版优先 Windows 桌面 | 用户要求桌面离线应用，当前工作环境为 Windows | 如果要支持 macOS/Linux，打包和测试范围扩大 |
